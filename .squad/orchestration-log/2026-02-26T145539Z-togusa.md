## Work Summary

**Problem:** `fontPipeline.test.ts` and `FontLoader.test.ts` crashed at suite load time with `TypeError: Cannot assign to read only property 'load'`. 0 tests ran; other 3 suites and 20 tests unaffected.

**Root cause:** opentype.js CJS/UMD bundle's factory function runs `exports.load = load` at module-evaluation time. Vitest's ESM strict mode seals the `exports` object → assignment throws.

**Solution:** Created `src/frontend/vitest.config.ts` with `resolve.alias: { 'opentype.js': '.../dist/opentype.module.js' }` to force ESM build resolution. Also added `@vitest-environment jsdom` to FontLoader.test.ts and `src/frontend/src/test-setup.ts` to stub jsdom-missing APIs.

**Result:** All 41 tests across 5 suites pass.
