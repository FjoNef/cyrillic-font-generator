### 2. Opset 17 → 18 auto-upgrade
- **Symptom:** PyTorch's new dynamo exporter only supports opset ≥18; silently upgrades opset 17 → 18
- **Fix:** Explicitly set `opset_version=18` in `export_onnx.py` (line 45)
- **Impact:** Deterministic opset, no surprise upgrades in future exports

### 3. INT8 quantization failure (opset 18 onnxruntime bug)
- **Symptom:** `onnxruntime.quantization.quantize_dynamic()` raises `ShapeInferenceError: Inferred shape and existing shape differ in dimension 0: (512) vs (256)`
- **Root cause:** Shape annotation conflict in opset 18 model weights (onnxruntime bug, not model architecture)
- **Workaround:** Wrapped quantization in try/except; falls back to fp32 (lines 68–82)
- **Impact:** Model exported as fp32 (230 MB) instead of INT8 target (~60 MB); structurally valid, inference confirmed working

### 4. External data sidecar consolidation
- **Symptom:** New torch dynamo exporter writes weights to `.data` sidecar file; original fallback left misnamed sidecar
- **Fix:** Reload with `onnx.load(load_external_data=True)` then save with `save_as_external_data=False` to produce single inline file (lines 84–87)
- **Impact:** Clean single-file export; no `.data` sidecar dependency
