### Integration Path ❌ CRITICAL GAP

**Backend serves:** `/api/model` OR `/models/v1/generator.onnx` (no `/api/models/v1/` route exists)  
**Impact:** In dev, Vite proxies `/api/*` to backend; request `/api/models/v1/generator.onnx` hits SPA fallback → returns HTML instead of binary. Same in production.  
**Fix required:** Change `App.tsx` + `ModelLoader.test.ts` to use `/api/model` endpoint.

### Backend Model Path Resolution ⚠️ MEDIUM
**Issue:** `ModelManifestCache` + `HandleModelDownload` resolve model path via `AppContext.BaseDirectory`, which at runtime is the binary output directory (e.g., `bin/Debug/net8.0/`), not repository root.  
**Impact:** Model file at `models/v1/generator.onnx` (repo root) will not be found unless copied to build output or path is absolute.  
**Fix required:** Either copy model to bin output directory during build, or configure `ModelPath` with absolute path for local dev.
